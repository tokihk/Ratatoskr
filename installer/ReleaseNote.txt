【アプリ更新情報】Ratatoskr 0.6.0
	☆主な機能追加/機能改善
	　・オプションダイアログで「Packet」ビューの色設定を行えるようにしました。

	　・Watch listで検出パケット数を表示するようにしました。

	　・Packet Converterバーを見やすくしました。
	　　(バーに情報表示、中身にインデント追加)

	　・About画面にPluginタブを追加。

	　・送信先が見つからないときは「Target alias」の背景色を黄色に表示するようにしました。
	　　緑 : 該当するAliasのGateが存在し、Connect状態のGateが存在する。
	　　黄 : 該当するAliasのGateが存在し、Connect状態のGateが存在しない。
	　　赤 : 該当するAliasのGateが存在しない。

	　・エディット送信モードに組み込み関数を追加。
	　　乱数データや繰り返しデータを送信できるようにしました。

	　　記述式例　　　　　　　　　　  送信データ
	　　02$(random-hex #1, #5)03　⇒ 02 (1～5バイトのランダムデータ) 03
	　　02$(repeat 112233, #2)03　⇒ 02 11 22 33 11 22 33 03

	　　詳細はヘルプを見てください。

	　・エディット送信モードで10進数データを扱えるようにしました。
	　　「#」で始めると10進数となります。
	    範囲は0～9999999です。

	　・フィルター式で時間定数を100m 100sec 100msec などで表現できるようにした。
	　　現状、整数表現にしか対応していません。
	　　0.5secは認識できません。500msecと記述してください。

	　・ファイルメニューからCSV保存したときにUTF-8変換したデータも出力するようにした
	　　変換できないデータは化けます。
	　　CSVを読込時、UTF-8部分のデータは無視します。

	　・複数種類のログを一度に読み込ませたときのロード速度を向上

	　・ログ読込中のパケット数を表示するようにした
	　　ログ読込中にキャッシュパケット数を超えた場合は、時系列で古い
	　　パケットから削除されてしまうため、キャッシュパケット数をいくつに
	　　設定すればいいかの目安にするために表示するようにしました。
	　　※キャッシュ/表示できるパケット数はオプションで設定できます。

	☆不具合修正
	　・「Packet」ビューの列タイプ「Data Preview(Binary)」で
	　　バイト間のスペースが表示されない不具合修正。
	　　「00 11 22 33」と表示したいのに「00112233」と表示される。

	　・Packet Viewのヘッダー設定画面でアイテムをウィンドウ範囲外までドラッグすると
	　　強制終了する不具合修正。

	　・パケット保存時に、保存ダイアログをキャンセルすると
	　　強制終了してしまう不具合修正。

	　・フィルター式で、括弧を使わないと演算子を組み合わせられなかった不具合修正。
	　　「HexString == "00" || HexString == "01" || HexString == "02"」のように
	　　３つ以上の条件を指定しようとしたら、
	　　「( HexString == "00" || HexString == "01" ) || HexString == "02"」
	　　のように括弧で囲む必要があった問題を修正。

	　・フィルター式で、「DeltaTime」が使えなかった不具合修正。

【アプリ更新情報】Ratatoskr 0.5.0
	☆主な機能追加/機能改善
	　・バイナリファイルの読込に対応
	　　.binのファイルはバイナリファイルとして認識します。
	　　1024バイト、10ms間隔のデータとして認識します。

	　・基本UIデザインを変更
	　　ぱっと見で全面に表示されるコントロールの角を丸くしました。
	　　パターン入力時のOK色を緑系に統一しました。

	　・Packetビューの列編集を拡張
	　　Packetビューの列名変更/タイプ変更に対応しました。
	　　また、列毎にPacket Filterを設定できるようにしました。

	　・Packetビューの右クリックメニューに「Export」を追加
	　　選択中パケットのデータ内容をファイルにExportできるように
	　　しました。

	　・Sequentialビューを高速化
	　　.NET Frameworkでの高速化に限界があったので、一部描画にWinAPIを使用
	　　するようにしました。

	　・Sequentialビューに行番号を追加
	　　デフォルト設定で、Sequentialビューに行番号が表示されるようになります。
	　　表示のON/OFFはメニュー「Tools - Option」の「System」から変更できます。

	　・Sequentialビューでメモリ制限を行えるようにした
	　　ビュー内でのメモリ使用量が一定を超えないように制限します。
	　　メモリ制限のON/OFFはメニュー「Tools - Option」の「System」から変更できます。

	☆不具合修正
	　・PacketViewのFilterの有効/無効のチェックが逆になっていたのを修正。

	　・[AudioFile]から発生したパケットが変化してしまう問題を修正

【アプリ更新情報】Ratatoskr 0.4
	☆主な機能追加/機能改善
	　・ファイル送信時の送信遅延に対応
	　　0～180000msまで設定できます。

	　　送信遅延は「データがセットできるようになってから」を基準とします。
	　　※最後に送信してからではありません。

	　・消費メモリ軽減

	　・キャッシュパケット数の表示に対応
	　　Ratatoskr内部では格納できるパケット数が決まっているので、
	　　大量の通信パケットを読み込ませた場合は古いパケットから
	　　削除します。

	　　フィルタ変更等で再描画等を行うと、キャッシュのパケット数しか
	　　再描画できないのですが、再描画されるパケット数を確認する術が
	　　なかったので確認できるようにしました。

	　・キャッシュパケット数/表示パケット数の設定変更に対応
	　　オプションから設定できます。99999～9999999まで。
	　　変更はパケットのクリアを行ったときに適用されます。
	　　※100万パケット以上はかなりメモリを消費しますので
	　　　取扱いにはご注意ください。

	　・フィルタに指定できるキーワードを更新
	　　  BinText => BinString
	      HexText => HexString
	      UnicodeLText => Utf16LeText
	      UnicodeBText => Utf16BeText

	　・フィルタに指定できるキーワードを追加
	      Class
	　　　ShiftJisText
	      EucJpText

	　・複数種類のログの同時読み込み対応

	　　v0.3以前でも複数のログをまとめてD&Dすることで複数ログを
	　　読み込ませることはできていましたが、それはファイル名を
	　　ソートして一つ目のログのフォーマットでそれ以降のログを
	　　順番に後ろに結合するだけのものでした。

	　　v0.4はファイル単位でログフォーマットを判断し、パケット単位で
	　　時刻ソートをかけながらログを結合します。

	　　※管理者モードの場合はWindows仕様によりD&Dができないので
	　　　ログ閲覧用にRatatoskrを別に起動するとかした方が
	　　　いいと思います。

	　・ディレクトリのD&Dに対応
	　　ディレクトリをD&Dすることでファイルを自動抽出して
	　　ファイルのオープン処理を行います。

	　・パケット変換機の表示/非表示に対応
	　　パケット変換器の上にバーがあるのでクリックしたら開閉します。

	　・パケット処理速度高速化
	　　CPU負荷を下げるための処理制限を少し緩和。
	　　CPU負荷も処理速度も当社比10倍。

	　・自動パケット保存の対象を選択できるようした
	　　デバイスから発生したパケットを保存するか、
	　　変換器を通った後のパケットを保存するかを
	　　選択できるようにしました。

	　・CSV保存時にローカル時刻も保存するようにした

	　・シリアルポートへの自動再接続
	　　接続中のシリアルポートが存在しなくなったときに
	　　自動で接続待機状態に移行するようにしました。

	　　USBシリアル等をつなぎ直すたびにポートオープンクローズを
	　　する必要がなくなります。

	　・シリアルポートが１つも存在しないときにSerial Portを
	　　選択すると強制終了してしまう不具合修正。

	　・スクリプトを実行しようとすると強制終了してしまう
	　　不具合修正。
	　　同梱しているアセンブリが足りなかったので同梱するファイルを
	　　追加しました。

	☆不具合修正
	　・Serial Portの設定画面を開くたびにParity/Stopbitが
	　　初期化されてしまう不具合修正
	　　※設定はできています。

	　・「Grouping - Receive timeout」で次のパケットの先頭も
	　　結合してしまう不具合修正。

	　・「Convert - Code Encode」が正しく動作しない不具合修正。

【アプリ更新情報】Ratatoskr 0.3.0
	◆バージョンアップ時の注意事項
	　・ログフォーマット(.gdcap)のバージョンをV000→V001に変更
	　　v0.3は古いバージョンで保存されたログも全て読み込む
	　　ことができますが、v0.2以前は新しいバージョンで保存した
	　　ログを読み込むことはできません。

	☆主な機能追加/機能改善
	　・マルチプロファイル対応
	　　右上のプロファイルリストでプロファイルを切り替えることができます。
	　　プロファイルのエクスポートは右上のプロファイルから行うように
	　　変更しています。

	　　プロファイルのインポートはD&DもしくはFile Openから行ってください。
	　　重複する名前のプロファイルが存在する場合は、名前の編集を強要します。

	　　※エクスポート→インポートを行うことでプロファイルのコピーが
	　　　できます。

	　・パケット処理高速化
	　　パケット構成を最適化した影響で旧バージョンのアプリでは
	　　ログが読み込めなくなります。
	　　（新しいバージョンは旧バージョンのログを読み込めます）

	　・スクリプト対応
	　　[MenuBar] -> [Tool] -> [Script Manager]を実行してください。
	　　スクリプトエディタが開きます。

	　　言語はC#です。
	　　APIや記述方法はドキュメントを参照してください。

	　・「Target alias」に全ゲートのAliasを固定リストとして追加。
	　　「Target alias」に「全ゲートのAlias」＋「履歴」が表示
	　　されるようになります。

	　・ショートカットキーでゲートの接続/切断が行えるようにした
	　　Ctrl＋1～5で切り替えられます。

	　・送信パネルに送信ボタンを追加。

	　・強制終了時にエラーログを保存。
	　　実行ファイルが置いている場所にエラーログ(abort.log)を出力します。
	　　何かあったときはこのファイルを要求するかもしれません。

	☆不具合修正
	　・自動パケット保存のフォーマットをrtcap設定にしているとき、
	　　ファイル切り替え時に過去に出力したパケットも一緒に
	　　出力されてしまう不具合修正。

	　・自動タイムスタンプを有効にできない不具合修正。
	　　（設定ファイル直接操作なら可能）
	　・自動パケット保存が動作しない不具合修正。
	　・プログレスバー制御時に強制終了することがある不具合修正。

【アプリ更新情報】Ratatoskr 0.2.0以前
	☆主な機能追加/機能改善
	　・プロファイルの保存/読込に対応。
	　　プロファイルにはGate設定、変換器、ビューの設定が含まれます。

	　　保存はメニューの「Export User Config」から行うことができます。
	　　読込はメニューのOpenか、ドラッグ＆ドロップで行うことができます。

	　　ユーザーデータの管理方法を変更しているため、対象データは初期化されます。

	　・パケットログの再生に対応
	　　送信データパネルから操作できます。

	　・Nullデバイスに対応
	　　特に何もしない仮想デバイスです。

	　・ヘルプファイル同梱
	　　docs内のindex.htmlを開いてください。

	　・1パケットあたりのメモリ消費量を削減

	　・保持パケット数に制限を追加
	　　メモリ枯渇による強制終了を防止するため、内部に保持するパケット数に
	　　制限を設けました。

	　　許容量を超えた場合は古いパケットから破棄されます。

	　・描画待機パケットが一定以上になったときに自動で高速描画モードに移行
	　　よっぽどのことがないかぎり発生しませんが、パケット発生速度よりも
	　　描画速度の方が遅いと、描画待機パケットが増加していきます。
	　　（Busyで表示されている数）

	　　描画待機パケット数が一定以上になったときに、自動で高速描画モードへ
	　　移行するようにしました。

	　　[高速描画モードについて]
	　　　かなり前のバージョンから対応しています。

	　　　パケットビューの更新を停止し、描画のスリープ処理を全て撤廃することで
	　　　パケットビューへパケットを高速転送します。
	　　　（通常モードの数倍の速度で描画します）
	　　　高速描画中は右下の「Norm」が「High」に変化します。

	　　　以下の条件で発動します。
	　　　・パケットログの読み込み
	　　　・パケット再描画
	　　　・右下の「Norm」をダブルクリック

	　　　描画待機パケットがなくなったら自動で通常モードに戻ります。

	　・ウォッチリスト対応
	　　フィルタ式と同じ構文でパケットを指定し、指定したパケットを検出した
	　　ときに通知を行います。
	　　※通知はEventとDialogに対応しています。

	　・通信レート表示に対応
	　　⇒ ゲート制御ボタンの下に通信レート表示を追加。
	　　　 通信レートをテキストとミニグラフで表示します。
	　　　 対象データは送信/受信/両方から選択できます。
	　　　 パケット変換は適用されません。
	　　⇒ ステータスバーに合算通信レート表示を追加。
	　　　 全ゲートの入出力を対象とした通信レートをテキスト表示します。
	　　　 対象データは送信/受信/両方から選択できます。
	　　　 パケット変換は適用されません。
	　　⇒ Packetビューの選択パケット情報に通信レートを追加。
	　　　 選択しているパケットの期間とデータサイズから算出します。
	　　　 パケット変換が適用されます。

	　・Sequentialビューの動作改善
	　　⇒ 表示高速化
	　　⇒ RichTextBoxの問題でパケットの切れ目によって無駄に
	　　　 改行されてしまう問題を回避
	　　⇒ EchoBack対応

	☆不具合修正
	　・パケットログを読み込んだときに時刻情報のミリ秒部分がおかしくなる不具合修正。
	　　ログファイル自体には問題ありません。
	　　読み込み時の不具合です。

	　・パケットビューのバイナリエディタにデータが表示されない不具合修正。

	　・フィルタ式でデコードできない正規表現を入力したときに強制終了する不具合修正。

	　・Packet Viewを表示したまま数百万パケット受信すると強制終了する不具合修正。
	　　（ガベージコレクションが期待通りに動作しない問題を修正）

	　・パケット保存時に最後のパケットが2回保存されてしまう問題を修正。
	　・CSV保存が行えない問題を修正。
	　・バイナリ保存が非常に遅かった問題を修正。
	　・自動保存機能が動作しない問題を修正。
